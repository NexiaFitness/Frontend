## Quick Start

### Prerequisites
- Node.js 20.19+ or 22.12+ (current: requires upgrade from 20.16.0)
- pnpm 8.0+

### Installation
```bash
# Clone repository
git clone https://github.com/NexiaFitness/Frontend.git
cd Frontend

# Install dependencies
pnpm install

# Start development server
cd apps/web
pnpm dev
```

The application will be available at `http://localhost:5173`

## Technical Architecture Decisions

### State Management: Redux Toolkit Only
**Decision:** Use RTK Query for all API communication, eliminating traditional service layers.

**Rationale:**
- Consistent pattern across web and future mobile apps
- Automatic caching, loading states, and error handling
- Type-safe integration with backend FastAPI
- Professional scalability for large applications

**Implementation:**
- вњ… `baseApi.ts` - Centralized API configuration with JWT injection
- вњ… `authApi.ts` - Authentication endpoints with auto-generated hooks
- вњ… Store configured with RTK Query middleware

### Authentication Flow
**Current Implementation:**
```typescript
// вњ… Available hooks (auto-generated by RTK Query)
import { 
  useLoginMutation,
  useRegisterMutation,
  useForgotPasswordMutation,
  useResetPasswordMutation,
  useGetCurrentUserQuery 
} from '@shared/api/authApi';
```

**Integration with Backend:**
- Base URL: `https://nexiaapp.com/api/v1`
- Automatic Bearer token injection in headers
- JWT persistence in localStorage
- Role-based access (Admin, Trainer, Athlete)
